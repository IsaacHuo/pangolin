# 第三章 基于零信任的智能体无感攻防演练平台架构设计

通过对模型通讯漏洞机理的梳理，本章将深入并系统化介绍用以支撑以上漏洞实控验证的综合Web应用系统“ Pangolin 智能体攻防靶场安全平台”的顶层选型方案及各块子模块结构职能以及如何拼串闭环流向拓扑的规划思路。

## 3.1 总体设计理念及前后端基础选型

作为一套需被安置在微服务极其密集且通讯实时长链接多路并发并置架构下环境的测试网关安全代理组件，整套项目的基础开发采用“强表现前端 + 极致性能解耦异步后端”进行严格的责任分解与全栈分离模式方案搭建。

- **零信任代理干预（MITM 透明中框隔离层原则）**：本着最大限度复原真机被入侵破坏的无打扰实验效果，本防备网关采用标准拦截中间件模式运作介入 L7 应用解析层监听器位；所有发生自靶标代理 Agent 发送或回撤的数据通讯需全部强行过轨穿越平台提供的甄别过滤引擎接受严格切面的断阻审查洗礼以及审计信息落档采集投送处理流程后才能予以上下释放传输以完成通信接驳动作。
- **底层核查引擎与网关中心架配选型（Backend Foundation）**：考量为负重吞载难以控制频率并且附带庞大数据内容的交互通信 JSON 包处理响应，后端选择使用 Python。并引入由于使用基于底层 asyncio及高性能 uvloop 网络运行调度圈而得到极大性能飞跃的架构级微服务外框体系 `FastAPI` 作为支点。不仅完成了纳秒极其低系统传输转换资源消耗需求更非常利于平台进行由于多任务隔离验证以及需要外借连接并等待超长轮询外部大型语义判决模型接口的多协异布验证功能支撑。
- **高实度全景交互响应展盘（Frontend Web Dashboard）**：作为提供给攻防研究开发操作人员的核心管控作业窗口工作前端产品端；它被设计要求能实时反馈无差别回拉所有网络包发生时间及隔离状态并在毫无界面感知凝滞阻塞度的情况下高刷展现。选定使用集成拥有目前业界最高动态驱动反映性能极快并配合底层 Vite 技术全打包渲染搭建能力的极其优秀前沿 `Nuxt 3` (基于 Vue 3 生态栈的核心)，以及 `TypeScript` 保障高度繁乱结构深嵌套类型的对等统一确保了界面呈现鲁棒可用稳定性呈现控制管理终端平台开发技术底库支持。

## 3.2 Pangolin 攻防验证平台的网络拓部结构链路

围绕以靶机服务沙盘实验测试数据流运转轨迹为逻辑推进轴线，系统总体构建呈现典型的倒漏斗筛选式的防御层次布局分为独立的四大功能群落子域。

### 3.2.1 模拟端接入代理与状态保持网（Protocol Proxy & State Maintainer）

面对智能模型多重通讯连接管道适配支持挑战环境（可能采用 SSE 单推送挂载长连接 HTTP 型抑或是直接以本基环境 Stdio 进行操作交互的情态），网闸在接入管口统配进行接驳解析无相融合封转还原操作行为支持能力模块开发。
不论目标请求数据帧是属于主动作呼寻操作请求亦或是下设环境反回读取的载荷。都均在管线端面截提取转码，最至关重的是配嵌建设有一个强壮生命管控自清回收状态控制字典（Session Stores）用于暂时接纳挂起所有通过而未过检的关联连接包（利用其包内协议唯一指定标号）；当审查流程尚未得出判断决断之前，管道底层必须严守原有与原生端口系统心跳保活通信；这种剥离拦截缓冲中介设计的核心便是用作让真实系统程序在使用本透明盾墙监控的同时杜绝发生意外丢断闪落系统连接而确保被防御方原程序的持久稳定无伤可用生存实验条件。

### 3.2.2 并发双引擎防越权拦截审核判官流水线 (Dual-Layer Intercept Pipeline)

位处于整座防卫建筑网关架构的心脏重重过滤把控制地重阵，此管组件负责决审与过滤阻挡处理。这套检验网络依据响应需求性能级层及逻辑抗阻难度深度切割设计成并路双网隔离运作关卡组合：

1. **L1 高速自动机静态字典特征屏蔽层（Static Analyser）**：此作为屏蔽低层次恶意攻击以及极大消耗降低总体性能冲击的防撞壁功能网络；配以汇集归编的各类高极重风险灾毁操作的指令名称序列字典并使用依赖底层 Rust 的 Aho-Corasick 复合有限状态自动检索机构构建查寻逻辑。面向拦截类似于读取 `/etc/hosts` 或者请求对源数据操作带有恶意抹断属性强标语（例如直接包含 `rm -rf /*` 或其它特定黑单风险代码执行特征段）时能几乎以纳秒极致瞬发时间点做果断定点直接隔离砍断封杀。
2. **L2 大语言动态意图认知评价过滤层（Semantic LLM Analyser）**：做为一套智能高级靶场对抗试验最至要的研究阵位点。其用于承装与抵消防备那类型刻意避免所有限制字段伪造打包了长自然模糊结构掩盖真实行为而穿透静态网兜直抵智能基座心核的语义注入伪攻击（Bypass Penetration）。网络将对拦截住提取下来的疑有长段落意向执行诱唤语句提送打压包发给外立的一个负责充当“警察”验证评价防备用高能大语模型驱动验证接口。它依靠专用预设置框架将通过的请求指令段要求大模型给出是否夹夹附意图攻击的判断理由结论与评分系数，根据界定分值的及回告结果布尔数值最后判决该动作指令下放执行亦或者是阻停异常报告原口返回结束。

### 3.2.3 表决阻断管理和深潜溯踪归档仓（Decision Controller & Audit Storage）

当截阻流管流水系统完成了所有预定防审程序评测后，综合管理裁决终端将会按照最苛刻（悲观极防制策略即只要一级防线出标红报告阻危全管拦截封存退绝报告原路发拒回应），它生成一个完美贴符原生架构错误规范的标准回包塞往原有管线以不造原生系统瘫亡处理阻接拒绝事件。
其同时具备极为重要的一个附职能，将请求全原本时间断点戳，从接至流，包括L1及特别是拥有推理原因判别的L2详细定标通过或者是截拒决因理由。所有大体量过程附记全通过纯异步低负担日志落档驱动组件按时许编织进入如极其严格难以抹消改变痕迹 JSONL 文件行内储存下压深地。供此后极难取理寻找的“跨越委托混乱攻击越层传递失控过程追踪”寻源和取究防漏洞漏失复盘提最绝对坚备的可信任原始日志大单支撑记录资料基础底座库房功能服务支撑提供服务体系支撑点建立和开发落地提供实处基础功能组件开发落地能力实现功能呈现展示作用成效呈现展示等呈现支撑构建功能支持系统和功能应用功能体现保障。

### 3.2.4 Web 全透多维实时监控展现推板中心 (Real-time Broadcast & Tracking Panel)

所有位于幕暗内处进行的判定搏杀拦截均会连接入配置有一条高畅通效率宽敞持久双开数据通道网络 WebSocket 的即时信息转发投射站网眼机进行转射发送至界端展示中心面。凭借着开发选设用以 Nuxt 响应重搭系统。所有的通过与隔止，每一次细微判定评分参数，和其附藏原协议请求全文本代码将无差无延迟直接跳呈瀑流图在总控仪战板监控盘内。不仅给予研究监控管理者宏达可视化掌控并且也能通过盘内界面即场动态调整关闭干涉各种诸如截打阻控严防阈门和模型驱动开关，甚至允许中停开启手工人工人审决放行模式等完全对安全防区干线系统的直接遥干预操作系统配置系统，构齐成为兼管检治演于同框的一套完满齐全体靶地功能中心界面产品交付展现。
